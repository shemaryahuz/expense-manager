## Expense Manager Server

Simple Express.js server that manages users, categories, and transactions using JSON files as a datastore.

### Tech stack
- **Runtime**: Node.js (ES modules)
- **Server**: Express 5
- **CORS**: Enabled for all origins
- **Storage**: JSON files under `database/`

---

## Getting Started

### Prerequisites
- Node.js 18+ recommended

### Installation
1. Clone this repository
2. Install dependencies

```bash
npm install
```

3. Start the server (default: port 3000)

```bash
npm start
```

You should see: `Server is running on port 3000`.

### Project Structure
```
expense-manager-server/
  database/
    users.json
    categories.json
    transactions.json
  src/
    controllers/
    routes/
  server.js
```

## Base URL
```
http://localhost:3000/api
```

All endpoints return JSON. Errors typically return a plain string message with appropriate HTTP status codes.

## Users API

Base path: `/api/users`

- GET `/` — Get all users
  - 200: Array of users
  - 404: "Users not found" if file exists but is empty array

- POST `/` — Create a user
  - Body (JSON):
    ```json
    {
      "id": "string",           // required for later deletes/lookup (client-generated)
      "name": "string",         // required
      "password": "string"      // required
    }
    ```
  - 200: Updated array of users
  - 400: "User name and password are required"

- DELETE `/:id` — Delete a user by id
  - 200: Updated array of users

Note: There is a route definition intended for getting a single user by id in the controller (`getUser` expects `req.params.id`), but the router currently does not expose `GET /:id`. Only `GET /` is defined in the router twice — the first handler (`getUsers`) will run. If you need single-user retrieval, update the router to include `router.get('/:id', getUser)`.

## Categories API

Base path: `/api/categories`

- GET `/:userId` — Get categories available to a specific user
  - Returns categories where `category.userId === userId` or `category.userId === null`
  - 404: "Categories not found" if none matched

- POST `/` — Create a category
  - Body (JSON):
    ```json
    {
      "id": "string",        // required for later deletes (client-generated)
      "name": "string",      // required
      "userId": "string"     // required (can be null in existing data, but POST requires it)
    }
    ```
  - 200: Updated array of categories
  - 400: "Category name and user id are required"

- DELETE `/:id` — Delete a category by id
  - 200: Updated array of categories

## Transactions API

Base path: `/api/transactions`

- GET `/:userId` — Get all transactions for a user (sorted desc by date)
  - 200: Array (may be empty)

- GET `/month/:userId/:year/:month` — Get a user's transactions for a given month

- GET `/category/:userId/:categoryId` — Get a user's transactions for a given category

- GET `/search/:userId?title=term` — Search by transaction title (case-insensitive substring match)

- POST `/` — Create a transaction
  - Body (JSON):
    ```json
    {
      "title": "string",       // required
      "type": "string",        // required (e.g., "expense" or "income")
      "amount": 123.45,        // required (number)
      "date": "YYYY-MM-DD",    // required (string, parseable by Date)
      "userId": "string",      // required
      "categoryId": "string"   // required
    }
    ```
  - 200: Created transaction object (id is generated by the server)
  - 400: "Transaction title, amount, user id and category id are required"

- DELETE `/:id` — Delete a transaction by id
  - 200: Deleted transaction object

## Example Requests

```bash
# Get all users
curl http://localhost:3000/api/users

# Create user
curl -X POST http://localhost:3000/api/users \
  -H "Content-Type: application/json" \
  -d '{
    "id": "u1",
    "name": "Alice",
    "password": "secret"
  }'

# Delete user
curl -X DELETE http://localhost:3000/api/users/u1

# Get categories for user
curl http://localhost:3000/api/categories/u1

# Create category
curl -X POST http://localhost:3000/api/categories \
  -H "Content-Type: application/json" \
  -d '{
    "id": "c1",
    "name": "Groceries",
    "userId": "u1"
  }'

# Get transactions for user
curl http://localhost:3000/api/transactions/u1

# Get transactions for user in May 2025
curl http://localhost:3000/api/transactions/month/u1/2025/5

# Get transactions for user and category
curl http://localhost:3000/api/transactions/category/u1/c1

# Search transactions by title
curl "http://localhost:3000/api/transactions/search/u1?title=mil"

# Create transaction
curl -X POST http://localhost:3000/api/transactions \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Milk",
    "type": "expense",
    "amount": 4.99,
    "date": "2025-01-01",
    "userId": "u1",
    "categoryId": "c1"
  }'

# Delete transaction
curl -X DELETE http://localhost:3000/api/transactions/t1
```

## Data Files

Data is stored in JSON files relative to `server.js`:
- `database/users.json`
- `database/categories.json`
- `database/transactions.json`

The server reads/writes these files on each request. Ensure the Node.js process has read/write permissions.

---

## Notes & Limitations
- No authentication or hashing is implemented; `password` is stored as plain text in JSON.
- IDs:
  - Users/Categories: clients should provide unique `id` values.
  - Transactions: `id` is generated by the server.
- Error responses are mostly plain text strings.
- The single-user GET endpoint is not wired in the router. If needed, change `src/routes/usersRouter.js` to include `router.get('/:id', getUser)`.
